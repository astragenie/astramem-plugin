## #33 — CLI project scope (silent operator ⇄ agent split)

### Current state (verified)
- `--project` / `--agent` flags **already exist** on both subcommands
  (added by FEAT-423 #20 / FEAT-424 #22):
  - `src/cli/remember.ts:50` parses `--project`, folds into `metadata.project` (`remember.ts:66`)
  - `src/cli/recall.ts:55` parses `--project`, sets **top-level** `project` on `RecallRequestSchema` (`recall.ts:73`)
- `RecallRequestSchema.project` exists (`src/contracts/wire.ts:50`), daemon
  exact-matches `provenance.project`.
- `IngestPayloadSchema` has **no top-level `project`** (`wire.ts:18-33`) — remember can only carry it inside `metadata`.

### Two real remaining gaps
1. **Write/read field asymmetry.** `remember` writes `metadata.project`;
   `recall` filters top-level `project` → `provenance.project`. These match
   **only if** the daemon maps ingest `metadata.project` → `provenance.project`.
   Must verify against the live daemon; if it does not map, that IS the silent
   split. → **Verification task before any code.**
2. **No shared default-project resolution (asks #2/#3 — the durable fix).**
   Confirmed: nothing in `src/` or `hooks/` reads `.claude/loop.json`
   `memory.project` or an `ASTRAMEM_PROJECT` env. The recall hook derives
   project from `basename $CWD` (`hooks/scripts/session-start-recall.sh:35`),
   which is NOT guaranteed to equal crew's `memory.project`. So CLI-with-no-flag
   and provider-in-a-repo still default to different scopes.

### Proposed change
- Add a `resolveProject()` helper (`src/lib/project.ts`): precedence
  `--project flag` → `ASTRAMEM_PROJECT` env → `.claude/loop.json` `memory.project` (walk up from cwd) → `basename(cwd)` fallback. Document the order.
- Wire `resolveProject()` into `remember`, `recall`, and the recall hook so all
  three converge on one scope inside a repo.
- Add `project` to `config get`/schema (`src/contracts/config.ts`) as an
  optional override (lowest precedence above basename).
- If daemon does NOT map `metadata.project`→`provenance.project`: add top-level
  `project` to `IngestPayloadSchema` and have remember set it (needs daemon
  coordination — flag as cross-repo dependency, do not silently diverge wire).

### Scope / risk
- Touches: `src/lib/project.ts` (new), `remember.ts`, `recall.ts`,
  `config.ts`, `session-start-recall.sh`. ~4 files + tests.
- Risk: wire-shape change (top-level project on ingest) is cross-repo — gate on
  daemon behavior verification. Everything else is client-local + safe.

## #35 — resolver backend identity (display-only)

### Current state (verified)
- `resolveProvider()` **already returns** `SelectorResult` with
  `providerName: 'local' | 'saas'` (`src/contracts/selector.ts:21`,
  `src/lib/selector.ts:182`) plus `source` and `latency_probe_ms`.
- `./selector` is exported (`package.json:12`).
- There is **no `RemoteHandle` type in this repo** — that name lives in the
  downstream shared `@astragenie/astramem-client`. In THIS repo the backend
  identity is already first-class on `SelectorResult`.

### Remaining gap
- Consumers that call `resolveProvider().provider` throw away the sibling
  `providerName`. The plugin has nothing more to expose — the field is already
  there. Gap is (a) documentation/discoverability and (b) an optional
  convenience so a caller holding only the provider handle can read its backend.

### Proposed change (small)
- Document `SelectorResult.providerName` as the supported backend-identity
  field in the `./selector` export doc + README.
- Optional: stamp `backend: providerName` onto the returned provider object (or
  add `resolveProviderVerbose()` alias) so downstream `@astragenie/astramem-client`
  0.2 can read it off the handle without threading `SelectorResult`.
- No wire change, no behavior change. Pure additive/exposure.

### Scope / risk
- Touches: `selector.ts`, `contracts/selector.ts`, README. ~2-3 files. Low risk.

## #34 — repo-visible MEMORY.md digest (genuinely new)

### Current state (verified)
- No markdown-export / digest helper exists in `src/` (investigator: 0 hits).
- SessionEnd hook shim exists (`hooks/scripts/session-end-summary.sh`) and
  follows the `exec bun bin/astramem ...` pattern — a clean insertion point.

### Proposed change
1. New subcommand `astramem export-md` (`src/cli/export-md.ts`):
   flags `--project` (via #33 `resolveProject()`), `--out`
   (default `.claude/astramem/MEMORY.md`), `--k`, `--types` (default
   `decision,lesson` — deliberate types only, per issue caution).
   Recall top-K per type → render markdown (one section/type, one bullet/atom:
   id, type, age, text) with a generated-file header noting daemon redaction +
   "edit via /remember".
2. Opt-in SessionEnd freshness hook variant: env `MEMORY_EXPORT_MD_ENABLE=1` +
   path override, **default OFF**; write-if-different only (no git churn on
   no-op sessions).
3. Doc the split vs SessionStart recall (#31/#32): recall = machine-local live
   session injection; MEMORY.md = committed human/CI snapshot.

### Dependencies / cautions
- Depends on #33 `resolveProject()` for correct default scope.
- Quality prerequisite: auto-distill low-signal atoms (see #28) would pollute a
  committed digest → default `--types` to deliberate types until the daemon
  durability gate (astramemory-local FEAT-441) lands. Ship filtered-by-default.
- **Never write without opt-in** — plugins committing files into user repos is
  trust erosion. Default OFF is non-negotiable.

### Scope / risk
- Touches: `src/cli/export-md.ts` (new), `bin/astramem` dispatch, new hook
  script, `hooks/hooks.json`, tests. ~4-5 files. Medium (new surface + hook).

## Architect-reviewer amendments (PASS-WITH-CONCERNS, 2 Critical)

- **[C1] resolveProject drift.** Do NOT parse crew's `.claude/loop.json` in v1 —
  no shared schema → silent revert to `basename(cwd)` if crew's format shifts.
  **v1 = flag → `ASTRAMEM_PROJECT` env → `config.project` → `basename(cwd)`.**
  Defer loop.json parse to v2 behind a contract test against crew's shape.
- **[C2] Third scoping channel.** `src/cli/ingest-transcript.ts:360` writes
  top-level `project_id` from its own `basename(cwd)` (`session-end-summary.sh:22-26`)
  — the highest-frequency provenance writer. **Wire `resolveProject()` here too**,
  else the split only narrows. Wiring list = remember, recall, recall-hook,
  **ingest-transcript + session-end/pre-compact/subagent shims**.
- **[M] Wire-shape enforcement.** Route the ingest-`project` decision through
  `src/lib/wire-probe.ts` (`atom@2` generation bump), not a one-shot probe —
  `IngestPayloadSchema` has no `wire_version`, so ad-hoc remap re-diverges silently.
- **[#34] Re-scrub at export time.** `scrub.ts` runs at ingest; atoms under an
  older/foreign scrub ruleset aren't re-scrubbed before git commit. Run scrub at
  `export-md` time — git is a higher-consequence sink than local recall.
- **[seq] #34 hard-blocker.** #34 transitively depends on #33's daemon-mapping
  probe *succeeding without cross-repo coordination*; if the probe finds a
  mismatch, #34 is blocked on an external timeline.
